# Карты на продакшене

## Как запустить карты на проде (кратко)

1. **Ключи Mapon** — в `.env` на сервере задать хотя бы один:
   ```env
   MAPON_API_KEY=ваш_ключ_из_кабинета_Mapon
   ```
   или по компаниям: `MAPON_API_KEY_LAKNA`, `MAPON_API_KEY_PADEKS`.

2. **Leaflet со своего домена** (чтобы не зависеть от unpkg):
   - На сервере после деплоя выполнить:
     ```bash
     npm ci
     npm run copy-leaflet
     ```
   - В `.env`:
     ```env
     MAPON_LEAFLET_LOCAL=true
     ```

3. **Тайлы карты** — по умолчанию браузер грузит их с OpenStreetMap. Если на проде это блокируют:
   - В `.env` включить прокси: `MAPON_TILE_USE_PROXY=true` (сервер должен иметь исходящий доступ к `tile.openstreetmap.org`).

4. Проверить: страница «Карта» (`/map`) и карта на странице трака (`/trucks/{id}`) открываются и показывают карту и маркеры.

Подробности и альтернативы — ниже.

---

## 1. Переменные окружения (.env)

На сервере в `.env` должны быть заданы ключи Mapon:

```env
# Базовый URL API (по умолчанию уже задан в config/mapon.php)
MAPON_API_BASE_URL=https://mapon.com/api/v1

# Ключи по компаниям (company_id => ключ из личного кабинета Mapon)
MAPON_API_KEY_LAKNA=ваш_ключ_компании_1
MAPON_API_KEY_PADEKS=ваш_ключ_компании_2

# Либо один общий ключ (если не используете keys по компаниям)
# MAPON_API_KEY=ваш_общий_ключ
```

Без этих ключей запросы к Mapon не пройдут: карта трака и страница «Карта» покажут «нет данных».

## 2. Сеть

- Сервер приложения должен иметь **исходящий доступ в интернет** к `https://mapon.com` (порты 443).
- Файрвол/прокси не должен блокировать запросы к Mapon API.

## 3. Если браузер блокирует unpkg.com или tile.openstreetmap.org

Карты подгружают Leaflet (JS/CSS) и тайлы карты. Если на проде блокируют внешние домены, есть два выхода.

### Вариант A: Leaflet и тайлы со своего домена

**Leaflet (без CDN):**

1. Установить Leaflet и скопировать в `public`:
   ```bash
   npm install leaflet --save-dev
   npm run copy-leaflet
   ```
2. В `.env` на проде:
   ```env
   MAPON_LEAFLET_LOCAL=true
   ```
   Тогда скрипт и стили берутся с вашего домена: `/vendor/leaflet/leaflet.js`, `/vendor/leaflet/leaflet.css`.

**Тайлы через прокси (браузер запрашивает только ваш домен):**

1. В `.env` на проде:
   ```env
   MAPON_TILE_USE_PROXY=true
   ```
2. Тайлы будут запрашиваться по адресу вида `https://ваш-домен/map/tiles/{z}/{x}/{y}.png`. Сервер сам ходит за картинками в OpenStreetMap и отдаёт их клиенту. Исходящий доступ с сервера к `tile.openstreetmap.org` должен быть разрешён.

После деплоя не забудьте выполнить `npm run copy-leaflet` на проде (или включить этот шаг в сборку), чтобы в `public/vendor/leaflet/` лежали `leaflet.js`, `leaflet.css` и папка `images/`.

### Вариант B: Разрешить внешние домены

Если можете настроить сеть/прокси/CSP, разрешите:

- `https://unpkg.com` — для Leaflet (если не используете `MAPON_LEAFLET_LOCAL=true`).
- `https://*.tile.openstreetmap.org` — для тайлов OSM (если не используете `MAPON_TILE_USE_PROXY=true`).

В CSP при необходимости добавьте эти домены в `script-src`, `style-src`, `img-src` / `connect-src`.

## 4. Альтернативные тайлы (другие карты)

По умолчанию используются тайлы OpenStreetMap. Можно переключиться на другой поставщик, задав в `.env` свой URL-шаблон (подставятся `{z}`, `{x}`, `{y}`):

```env
# Пример: светлая карта CartoDB (без ключа)
MAPON_TILE_LAYER_URL=https://cartodb-basemaps-a.global.ssl.fastly.net/light_all/{z}/{x}/{y}.png
MAPON_TILE_ATTRIBUTION=&copy; <a href="https://carto.com/">CARTO</a>
```

Другие варианты (проверяйте лицензии и лимиты):

| Провайдер        | Комментарий |
|------------------|-------------|
| **OpenStreetMap** | По умолчанию, бесплатно. |
| **CartoDB** (Light/Dark) | Бесплатный тайр, без ключа. |
| **Stadia Maps**   | Есть бесплатный тариф, нужна регистрация и ключ. |
| **Mapbox**        | Бесплатный лимит, нужен токен. |
| **Yandex**        | Отдельный JS API, не только тайлы; для Leaflet можно искать тайлы Yandex (шаблон URL и лицензия у них свои). |

Если нужна **другая карта целиком** (не Leaflet + тайлы), например Yandex Maps или 2GIS, это уже замена фронтенда: другой JS API и разметка. Текущий код рассчитан на Leaflet; смену провайдера тайлов через `.env` выше достаточно для смены «подложки» карты.

## 5. Переменные .env для карт (сводка)

| Переменная | Описание |
|------------|----------|
| `MAPON_LEAFLET_LOCAL` | `true` — брать Leaflet из `public/vendor/leaflet/` (после `npm run copy-leaflet`). |
| `MAPON_LEAFLET_JS_URL` | URL скрипта Leaflet (если не локальный). |
| `MAPON_LEAFLET_CSS_URL` | URL стилей Leaflet (если не локальный). |
| `MAPON_TILE_USE_PROXY` | `true` — тайлы отдаются через маршрут `/map/tiles/{z}/{x}/{y}.png` с вашего домена. |
| `MAPON_TILE_LAYER_URL` | Шаблон URL тайлов (`{z}`, `{x}`, `{y}`). При включённом прокси не используется. |
| `MAPON_TILE_ATTRIBUTION` | Текст/HTML атрибуции карты (подвал на карте). |

## 6. Кэш

Данные Mapon кэшируются на 2 минуты (страница «Карта») и на 5 минут (блок на странице трака). Используется тот же драйвер кэша, что и в приложении. Отдельная настройка для карт не нужна.

## 7. Безопасность (из REFACTORING_PLAN)

- Ключи Mapon не должны попадать в логи и стектрейсы.
- В ErrorReport не включайте полный `.env`.

## Краткий чеклист

| Что проверить | Где |
|--------------|-----|
| Ключи Mapon в `.env` | Сервер |
| Исходящий HTTPS до mapon.com | Сервер / сеть |
| При блокировке CDN: `MAPON_LEAFLET_LOCAL=true` и `npm run copy-leaflet` | Прод |
| При блокировке тайлов: `MAPON_TILE_USE_PROXY=true` | .env на проде |
| Сайт по HTTPS | Конфиг веб-сервера |

После деплоя откройте страницу трака с заполненным `mapon_unit_id` и страницу «Карта» — должны подгружаться данные Mapon и отображаться маркеры.
