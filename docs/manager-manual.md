# Руководство менеджера — Fleet Manager

Инструкция по работе с веб‑приложением Fleet Manager: все разделы и основные действия для пользователя с полным доступом.

---

## 1. Вход и навигация

- Вход в систему выполняется по **email и паролю** (учётная запись создаётся администратором).
- После входа открывается **главная страница (Dashboard)**.
- Слева расположено **боковое меню**:
  - **Dashboard** — сводка по истекающим документам
  - **Водители** — список и карточки водителей
  - **Транспорт** — подменю: **Тягачи**, **Прицепы**
  - **Клиенты** — база клиентов (компании)
  - **Рейсы** — список, создание, просмотр и редактирование рейсов
  - **Статистика** — подменю: **Обзор**, **События одометра**
  - **Счета** — список счетов и платежи
- В шапке: приветствие с именем пользователя и кнопка **Выйти**.

На мобильных устройствах меню открывается по кнопке «гамбургер» (☰).

---

## 2. Dashboard (главная)

На главной странице отображается **таблица истекающих документов** (срок действия в течение ближайших 30 дней).

- **Типы сущностей**: водители, тягачи, прицепы, грузы (TripCargo).
- **Документы**:
  - у **водителей**: права (License), Code 95, разрешение (Permit), медсправка (Medical), декларация (Declaration);
  - у **тягачей**: техосмотр (Inspection), страховка (Insurance), техпаспорт (Tech passport);
  - у **прицепов**: техосмотр, страховка, TIR, техпаспорт;
  - у **грузов** — при необходимости свои сроки.
- Доступны **поиск**, **сортировка** по столбцам и **количество строк** на странице.
- По каждой строке видно: тип, название (ФИО водителя / марка и госномер ТС и т.д.), документ, дата окончания, сколько дней осталось, компания, статус.

Используйте Dashboard, чтобы вовремя продлевать документы водителей и транспорта.

---

## 3. Водители

- **Список водителей** — таблица с поиском и сортировкой.
- **Добавить водителя** — кнопка «Создать» → форма: имя, фамилия, контакты, документы (права, Code 95, медсправка и т.д.), компания, PIN для входа в приложение водителя (4–6 цифр).
- **Карточка водителя** — просмотр всех данных, документов, привязок.
- **Редактирование** — изменение данных и документов.
- При необходимости в карточке/редактировании можно **удалить** водителя (если это предусмотрено интерфейсом).

PIN водителя нужен ему для входа в мобильное приложение (раздел «Водитель»).

---

## 4. Транспорт

### Тягачи

- **Список тягачей** — таблица с основными данными (марка, модель, госномер, статус, компания).
- **Добавить тягач** — форма: марка, модель, госномер, компания, даты документов (техосмотр, страховка, техпаспорт), при необходимости — наличие CAN (для автоматического одометра).
- **Карточка тягача** — полные данные, документы, история.
- **Редактирование** — изменение данных и сроков документов.

### Прицепы

- **Список прицепов** — таблица (марка, госномер, тип, компания и т.д.).
- **Добавить прицеп** — тип (в т.ч. контейнерный), марка, госномер, документы (техосмотр, страховка, TIR, техпаспорт).
- **Карточка** и **редактирование** — по аналогии с тягачами.

Транспорт привязывается к рейсам при создании или редактировании рейса.

---

## 5. Клиенты

- **Список клиентов** — компании (клиенты, грузоотправители, грузополучатели).
- **Добавить клиента** — название, страна, город, адрес, реквизиты, контакты.
- **Карточка** и **редактирование** — просмотр и изменение данных.

Клиенты используются в рейсах как заказчики (customer), грузоотправители (shipper) и грузополучатели (consignee) по грузам (cargos).

---

## 6. Рейсы

### Список рейсов

- **Рейсы** — таблица с фильтрами и сортировкой (по дате, статусу, водителю и т.д.).
- Из списка можно перейти к **созданию** рейса, **просмотру** или **редактированию** выбранного рейса.

### Создание рейса

Пошагово заполняются данные:

1. **Экспедитор** — выбор компании (expeditor) и банка для реквизитов.
2. **Перевозчик**:
   - свой транспорт (экспедитор или другая внутренняя компания);
   - либо **третья сторона (third party)** — тогда создаются новая компания, тягач и прицеп, а в рейс добавляется расход на оплату субперевозчику.
3. **Транспорт** — водитель, тягач, прицеп (или данные третьей стороны).
4. **Маршрут (шаги)** — точки погрузки и разгрузки:
   - тип шага: погрузка (loading) / разгрузка (unloading);
   - страна, город, адрес, дата и время;
   - порядок шагов можно менять (drag-and-drop при редактировании).
5. **TIR / таможня** — при международных перевозках можно включить таможенное оформление (TIR) и указать **TIR пограничный пункт** (адрес пункта пропуска/таможни).
6. **Грузы (cargos)** — один или несколько:
   - клиент (customer), грузоотправитель (shipper), грузополучатель (consignee);
   - цена фрахта, НДС;
   - **позиции груза (items)**: описание, упаковки, паллеты, вес (нетто/брутто), объём, погрузочные метры, при необходимости — опасный груз, температура, код ТН ВЭД (customs_code) и т.д.;
   - привязка груза к **шагам погрузки и разгрузки** (какие шаги маршрута относятся к этому грузу).
7. Для контейнерных рейсов — номер контейнера и пломбы (если используются в форме).

После сохранения создаются рейс, шаги маршрута, грузы и позиции; при third party — компания, тягач, прицеп и расход.

### Просмотр рейса

На странице отображаются:

- **Шапка**: номер рейса, статус, схема (свой/третья сторона), экспедитор, перевозчик, транспорт (водитель, тягач, прицеп), при third party — фиксированная оплата.
- **Сводные показатели**: общий вес, объём, погрузочные метры, фрахт с НДС/без, стоимость товара, сумма по счетам поставщика и т.д.
- **Редактор маршрута** (аккордеон «Маршрут») — компонент **Trip Route Editor**: изменение порядка шагов перетаскиванием, сохранение нового порядка.
- **Грузы по клиентам** — группы грузов с развёрнутыми данными:
  - по каждому грузу: отправитель → получатель, упаковки, паллеты, вес, объём, фрахт, стоимость товара, счёт поставщика;
  - точки маршрута, связанные с этим грузом (погрузка/разгрузка), с датой и временем.
- **Документы по грузу**:
  - **CMR** — номер, кнопка «Сгенерировать CMR»: создаётся PDF и привязывается к грузу;
  - **Счёт (Invoice)** — генерация счета по грузу, сохранение PDF.
- **Расходы по рейсу** — список расходов (топливо, платные дороги, парковка и т.д.), при необходимости добавление/редактирование со стороны менеджера (если реализовано в интерфейсе).

Водитель выполняет рейс в мобильном приложении (статусы шагов, одометр, загрузка документов, расходы). Менеджер здесь контролирует маршрут, грузы и документы.

### Редактирование рейса

Те же блоки, что и при создании: экспедитор, перевозчик, транспорт, шаги маршрута, грузы и позиции. Изменения сохраняются в существующий рейс. Порядок шагов удобнее менять в **просмотре рейса** через редактор маршрута.

---

## 7. Статистика

### Обзор

- Таблица по рейсам за выбранный период: даты, водитель, тягач, пробег, расходы и т.д.
- Фильтры по периоду (дата от/до, быстрые диапазоны), поиск, сортировка.
- Используется для анализа рейсов и пробега.

### События одометра

- Таблица событий одометра: выезд/возврат в гараж, события по шагам маршрута, расходы водителя (топливо, AdBlue) с привязкой к одометру.
- Фильтры: тип события, водитель, тягач, период (дата от/до).
- Полезно для проверки корректности пробега и привязки расходов к километражу.

---

## 8. Счета (Invoices)

- **Список счетов** — таблица с номерами, датами, суммами, статусом оплаты (оплачен/частично/не оплачен).
- Фильтры: поиск, статус (все / оплачен / частично / не оплачен), сортировка.
- **Открыть PDF** — просмотр или скачивание сгенерированного счета.
- **Внести платёж** — дата и сумма платежа по выбранному счёту; сумма остатка пересчитывается.

Счета по грузам создаются из карточки рейса (кнопка генерации счёта по грузу).

---

## 9. CMR и счета по рейсу

- **CMR** генерируется в **просмотре рейса** по каждому грузу: кнопка «Сгенерировать CMR» → создаётся PDF, сохраняется и привязывается к грузу. Номер CMR можно задать/редактировать в форме.
- **Счёт (Invoice)** по грузу генерируется там же: кнопка генерации счёта → создаётся PDF счёта, запись появляется в разделе **Счета**. Дальнейшая работа с оплатой — в разделе «Счета».

---

## 10. Профиль и выход

- **Профиль** — просмотр и при необходимости изменение своих данных (если реализовано).
- **Выйти** — кнопка в меню в шапке справа; завершает сессию и перенаправляет на страницу входа.

## Краткая памятка по разделам

| Раздел | Основные действия |
|--------|--------------------|
| **Dashboard** | Контроль истекающих документов водителей, тягачей, прицепов |
| **Водители** | Список, создание, карточка, редактирование, PIN для приложения |
| **Тягачи** | Список, создание, карточка, редактирование, документы, CAN |
| **Прицепы** | Список, создание, карточка, редактирование, тип (в т.ч. контейнер) |
| **Клиенты** | Список, создание, карточка, редактирование компаний |
| **Рейсы** | Список, создание (экспедитор, перевозчик, транспорт, шаги, грузы), просмотр (маршрут, грузы, CMR, счета, расходы), редактирование |
| **Статистика → Обзор** | Рейсы за период, пробег, расходы |
| **Статистика → События** | События одометра, фильтры по водителю/тягачу/типу/дате |
| **Счета** | Список счетов, открытие PDF, внесение платежей |
| **В рейсе** | Редактор маршрута (порядок шагов), генерация CMR и счёта по грузу |

## Связь с приложением водителя

- Водитель входит в **мобильное приложение** по **PIN** (назначается в карточке водителя).
- В приложении водитель: отмечает выезд/возврат в гараж с одометром, меняет статусы шагов (погрузка/разгрузка), загружает документы по шагам, добавляет расходы по рейсу.
- Менеджер в веб‑интерфейсе видит рейсы, маршрут, грузы, сгенерированные CMR и счета, расходы и события одометра в статистике. При необходимости маршрут можно скорректировать (редактор маршрута в просмотре рейса).
