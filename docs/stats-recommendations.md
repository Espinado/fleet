# Рекомендации по расширению статистики для менеджеров и владельцев

Документ описывает, какую статистику можно добавить в Fleet Manager и как аналогичные метрики используют крупные логистические компании (DHL Freight, DB Schenker, Geotab, Fleetio и др.).

---

## Что уже есть в Fleet Manager

| Раздел | Метрики |
|--------|--------|
| **Статистика → Обзор** | Рейсы за период, фрахт, расходы, прибыль по рейсу, сводка (кол-во рейсов, суммарный фрахт/расходы/прибыль, средняя маржа %), график по неделям/месяцам (фрахт, прибыль) |
| **Статистика → События одометра** | События выезд/возврат, шаги маршрута, расходы водителя; сводка по категориям (топливо, AdBlue, литры, суммы); экспорт в PDF |
| **Dashboard** | Истекающие документы (водители, тягачи, прицепы, грузы) |

---

## Категории KPI в логистике (по отраслевым практикам)

Крупные операторы (DHL, Geotab, Fleetio) выделяют четыре блока:

1. **Финансы и себестоимость** — стоимость за км/рейс, маржа, дебиторка.
2. **Утилизация и загрузка** — загрузка транспорта и водителей, пробег, рейсы на единицу.
3. **Эффективность и сроки** — своевременность доставки, задержки, простои.
4. **Безопасность и соответствие** — документы, инциденты (если будут данные).

Ниже — идеи с опорой на уже имеющиеся в системе сущности (Trip, TripCargo, TripExpense, Invoice, Driver, Truck, Client).

---

## 1. Финансы и себестоимость

### Уже есть
- Фрахт, расходы, прибыль по рейсу и в сводке.
- Средняя маржа % по отфильтрованным рейсам.

### Что добавить

| Метрика | Описание | Источник данных | Сложность |
|--------|----------|------------------|-----------|
| **Стоимость за км (cost per km)** | Общие расходы рейса / пробег (км). Позволяет сравнивать рейсы и тягачи. | `TripExpense` (без subcontractor), пробег из одометра или `odo_end_km - odo_start_km` | Низкая |
| **Расходы по категориям** | Топливо, AdBlue, платные дороги, штрафы, ремонт и т.д. за период (в деньгах и в % от общих расходов). | `TripExpense` + `TripExpenseCategory` | Низкая (частично есть в «Событиях») |
| **Дебиторская задолженность** | Сумма неоплаченных/частично оплаченных счетов, средний срок просрочки. | `Invoice`, `InvoicePayment`, `due_date` | Низкая |
| **Выручка/прибыль по клиенту (customer)** | Фрахт и прибыль по клиентам (customer_id в TripCargo). Топ клиенты по объёму. | `TripCargo` (customer_id, price), расходы по рейсу | Средняя |
| **Выручка по экспедитору** | Фрахт по компаниям-экспедиторам (expeditor_id). | `Trip` (expeditor_id), `TripCargo.price` | Низкая |

Реализация: отдельная страница «Статистика → Финансы» или блоки на дашборде: сводные карточки (дебиторка, cost per km за период) + таблица/график по клиентам и по категориям расходов.

---

## 2. Утилизация транспорта и водителей

| Метрика | Описание | Источник данных | Сложность |
|--------|----------|------------------|-----------|
| **Пробег по тягачу** | Суммарный пробег за период по каждому тягачу (из событий одометра или `odo_end - odo_start` по рейсам). | `TruckOdometerEvent`, `Trip` (truck_id, odo_*_km) | Низкая |
| **Количество рейсов по тягачу/водителю** | Рейсы за период по технике и по водителям. | `Trip` (truck_id, driver_id), даты | Низкая |
| **Утилизация по дням** | Доля дней с хотя бы одним рейсом от общего числа дней в периоде (по парку или по тягачу). | `Trip` (start_date, truck_id) | Средняя |
| **Средний пробег на рейс** | По парку или по тягачу/водителю. | Пробег из одометра/рейса, `Trip` | Низкая |

Реализация: страница «Статистика → Транспорт и водители» с фильтром по периоду: таблица (тягач / водитель, кол-во рейсов, пробег, средний пробег на рейс) и при желании — простой график «рейсы по дням» или «пробег по тягачам».

---

## 3. Топливо и расходы на эксплуатацию

| Метрика | Описание | Источник данных | Сложность |
|--------|----------|------------------|-----------|
| **Расход л/100 км** | Литры топлива (и при необходимости AdBlue) / пробег * 100. По рейсу, по тягачу, по периоду. | `TripExpense` (category=fuel, liters), пробег рейса | Низкая |
| **Стоимость топлива за км** | Сумма по топливу / пробег. | То же + amount | Низкая |
| **Сравнение тягачей по расходу** | Таблица: тягач, пробег, литры, л/100 км, сумма по топливу. | `Trip` + `TripExpense` (fuel/adblue) + пробег | Средняя |

В «Событиях одометра» уже есть сводка по литрам и суммам по категориям; имеет смысл вынести «топливную» аналитику в отдельный блок или страницу «Статистика → Топливо» с разрезом по тягачам и периоду.

---

## 4. Своевременность и задержки

| Метрика | Описание | Источник данных | Сложность |
|--------|----------|------------------|-----------|
| **Рейсы с задержкой** | Доля рейсов, где по грузу указана задержка (has_delay) или delay_days > 0. | `TripCargo.has_delay`, `delay_days` | Низкая |
| **Сумма штрафов/компенсаций по задержкам** | Суммарное delay_amount по грузам за период. | `TripCargo.delay_amount` | Низкая |
| **Среднее время рейса** | Разница между датой/временем окончания и начала (по рейсу или по шагам). | `Trip.started_at`/`ended_at` или события одометра | Средняя |

Реализация: блок на дашборде или раздел «Статистика → Качество»: количество и % рейсов с задержкой, сумма delay_amount, при наличии — средняя длительность рейса.

---

## 5. Клиенты и контрагенты

| Метрика | Описание | Источник данных | Сложность |
|--------|----------|------------------|-----------|
| **Топ клиенты по фрахту** | Клиенты (customer) с сортировкой по сумме price за период. | `TripCargo` (customer_id, price) | Низкая |
| **Топ отправители/получатели** | Аналогично по shipper_id / consignee_id. | `TripCargo` (shipper_id, consignee_id) | Низкая |
| **Количество грузов/рейсов по клиенту** | Сколько рейсов и грузов пришлось на клиента. | `TripCargo`, `Trip` | Низкая |

Удобно сделать страницу «Статистика → Клиенты» с таблицей и фильтром по периоду: клиент, кол-во рейсов/грузов, сумма фрахта, при желании — средний чек.

---

## 6. Дашборд для владельца (сводный экран)

Имеет смысл вынести на один экран ключевые цифры для владельца:

- **День/неделя/месяц**: выручка (фрахт), расходы, прибыль, количество рейсов.
- **Дебиторка**: сумма неоплаченных счетов, кол-во просроченных.
- **Документы**: количество истекающих в ближайшие 7/14/30 дней (уже есть на Dashboard, можно дублировать карточкой).
- **Топ-3 клиента** за период по фрахту.
- **Топ-3 тягача** по пробегу или по рейсам.
- **Задержки**: число рейсов с задержкой и сумма delay_amount за период.

Реализация: отдельная страница «Дашборд владельца» или расширение текущего Dashboard блоками с агрегатами (запросы к Trip, TripCargo, Invoice, TripExpense с фильтром по периоду и по компаниям пользователя).

## Как это делают крупные логистические компании

- **DHL Freight** (KPI): точность доставки (без инцидентов/задержек), время доставки (on-time), утилизация парка (объём/вес), стоимость перевозки за единицу.  
  В Fleet Manager этому соответствуют: учёт задержек (TripCargo.has_delay, delay_days, delay_amount), даты рейсов и при желании — расчёт доли «в срок»; расходы и фрахт уже есть — можно добавить cost per km и разрез по клиентам.

- **DB Schenker (Ocean Bridge)**: мониторинг в реальном времени, прогнозы ETA, раннее выявление исключений.  
  Для автопарка это можно упростить до: список активных рейсов, статусы шагов, предупреждения по просроченным документам (уже есть) и по просроченным счетам (дебиторка).

- **Geotab / Fleetio**: cost per mile, расход топлива (MPG/л/100 км), утилизация, поведение водителей (езда, простои), профилактическое обслуживание.  
  В нашей системе: cost per km и л/100 км — естественное расширение имеющихся расходов и одометра; утилизация — по рейсам и дням; «поведение» пока без телематики можно не делать.

- **Fleetio**: метрики по TCO, ремонтам, осмотрам.  
  При появлении учёта ремонтов и осмотров по ТС те же отчёты можно добавить и у нас.

## Приоритеты внедрения

| Приоритет | Что делать | Зачем |
|-----------|------------|--------|
| 1 | Cost per km по рейсам и в сводке за период | Сравнение рейсов и тягачей, контроль себестоимости |
| 2 | Дебиторка: неоплаченные счета, просрочка | Управление денежным потоком |
| 3 | Топ клиентов по фрахту за период | Фокус на ключевых заказчиках |
| 4 | Пробег и кол-во рейсов по тягачу/водителю | Утилизация парка и планирование |
| 5 | Расход л/100 км (топливо) по рейсу/тягачу | Контроль расхода топлива |
| 6 | Рейсы с задержкой и сумма delay_amount | Качество перевозок и штрафы |
| 7 | Сводный дашборд для владельца | Один экран с главными KPI |

Дальше можно добавлять разрезы по экспедиторам, по категориям расходов на отдельной странице и при появлении данных — безопасность и TCO.
