# План углублённого тестирования: рейсы, заказы, документы

Полное и углублённое тестирование проектов **Fleet Manager** и **Cargo Trans** с фокусом на создание и редактирование рейсов, связанные документы и таблицы. Рейсы создаются **комбинированно**: из заказов (одним или несколькими) и/или ручным вводом данных.

---

## 1. Области тестирования

| Область | Что проверяем |
|--------|----------------|
| **Создание рейса** | Ручной ввод; из одного заказа; из нескольких заказов; смешанное (заказы + ручные шаги/грузы). |
| **Редактирование рейса** | Изменение маршрута, грузов, транспорта, дат; сохранение без потери связей. |
| **Заказы в рейсе** | Добавление заказов к существующему рейсу; удаление заказа из рейса; обновление маршрута и грузов. |
| **Документы** | CMR, Invoice, Transport Order (PDF); документы по шагам и общие по рейсу (загрузка, отображение). |
| **Таблицы и списки** | Списки рейсов/заказов; фильтры; пагинация; корректность отображаемых полей. |

---

## 2. Создание рейсов

### 2.1 Ручное создание (без заказов)

- [ ] Открыть `/trips/create` без предзаполнения из заказа.
- [ ] Заполнить обязательные поля: экспедитор, водитель, тягач, прицеп, даты, валюта.
- [ ] Добавить минимум один шаг маршрута (погрузка) и один (разгрузка), указать страну/город/адрес/дату-время.
- [ ] Добавить минимум один груз: заказчик, отправитель, получатель, привязать к шагам погрузки/разгрузки, цена, НДС.
- [ ] Добавить позиции груза (описание, паллеты, вес, объём и т.д.).
- [ ] Сохранить рейс — редирект на просмотр рейса, в списке рейсов новая запись.
- [ ] Проверить: маршрут (Reisa maršruts) отображает все шаги; блок «Kravas» — все грузы и позиции; даты начала/конца соответствуют введённым.

**Граничные случаи:**

- [ ] Попытка сохранить с разгрузкой по порядку раньше погрузки — должна быть ошибка валидации.
- [ ] Несколько грузов с разными парами шагов (L→U, L→U) — порядок шагов корректен.
- [ ] Контейнер/пломба, TIR/таможня — поля сохраняются и отображаются.

### 2.2 Создание из одного заказа

- [ ] Иметь заказ в статусе «Подтверждён» (confirmed), без `trip_id`.
- [ ] На странице заказа нажать «Создать рейс» (или перейти к созданию рейса с предзаполнением по заказу).
- [ ] На `/trips/create` выбрать заказ в блоке «Izvēlēties pasūtījumus» (или перейти из карточки заказа с уже выбранным заказом).
- [ ] Применить заказ — подставляются: экспедитор, заказчик, шаги маршрута (погрузка/разгрузка), грузы (отправитель/получатель, описание, вес, объём и т.д.), даты.
- [ ] Проверить: все точки маршрута на месте; город из заказа подставлен в шаг (если был указан или определён из адреса).
- [ ] При необходимости дописать вручную: водитель, тягач, прицеп, скорректировать даты/адреса.
- [ ] Сохранить — рейс создан, заказ перешёл в статус «Pārvērts reisā» (converted), привязан к рейсу (`trip_id`).
- [ ] На просмотре рейса: в блоке «Pasūtījumi reisā» отображается этот заказ; маршрут и грузы соответствуют заказу.

### 2.3 Создание из нескольких заказов

- [ ] На `/trips/create` в селекте «Izvēlēties pasūtījumus» выбрать несколько подтверждённых заказов (Ctrl+клик).
- [ ] Нажать «Pielietot» / применить — шаги и грузы всех заказов объединяются (не заменяют друг друга).
- [ ] Проверить: в блоке «Piesaistīts N pasūtījums(-i)» отображаются все выбранные заказы; маршрут содержит все точки погрузки/разгрузки по каждому заказу; количество грузов = сумма грузов по заказам.
- [ ] Убедиться: при добавлении ещё одного заказа по одному он добавляется к уже привязанным (не заменяет).
- [ ] Удалить один заказ из привязанных (кнопка «Noņemt no reisa» с подтверждением) — его шаги и грузы убираются из формы; заказ снова доступен в селекте.
- [ ] Сохранить рейс — все оставшиеся заказы в статусе converted, привязаны к рейсу; в «Pasūtījumi reisā» все они отображаются.

### 2.4 Смешанное создание (заказы + ручные правки)

- [ ] Применить один или несколько заказов.
- [ ] Вручную добавить ещё один шаг маршрута или ещё один груз.
- [ ] Изменить даты/адреса у подставленных шагов.
- [ ] Сохранить — все данные (из заказов + ручные) сохранены, связи шаг–груз корректны.

## 3. Редактирование рейса

- [ ] Открыть существующий рейс (со статусом не «Завершён»), нажать «Labot» → `/trips/{id}/edit`.
- [ ] Изменить даты начала/конца, добавить/удалить шаг маршрута, изменить адрес или порядок шагов.
- [ ] Добавить или удалить груз; изменить привязку груза к шагам погрузки/разгрузки.
- [ ] Сохранить — изменения отображаются на странице просмотра рейса и в блоке «Reisa maršruts».
- [ ] Рейс с привязанными заказами: после редактирования маршрута/грузов блок «Pasūtījumi reisā» и список грузов по-прежнему корректны (без дублей и потерь).

---

## 4. Добавление и удаление заказов в существующем рейсе

### 4.1 Добавление заказов

- [ ] Открыть рейс (не завершённый), в блоке «Izvēlēties pasūtījumus» выбрать один или несколько доступных заказов.
- [ ] Нажать «Pievienot pasūtījumus reisam».
- [ ] Проверить: в «Pasūtījumi reisā» появились новые заказы; в «Reisa maršruts» добавились точки маршрута из этих заказов; количество грузов и список клиентов обновились (блок «Kravas»).
- [ ] Убедиться: добавленные заказы больше не отображаются в селекте выбора (уже привязаны).

### 4.2 Удаление заказа из рейса

- [ ] В блоке «Pasūtījumi reisā» нажать «Noņemt no reisa» у одного из заказов (без подтверждения или с тостом — по текущей реализации).
- [ ] Проверить: заказ исчез из списка «Pasūtījumi reisā»; маршрут обновился — шаги, принадлежавшие только этому заказу, удалены; грузы этого заказа удалены из «Kravas»; счётчики и списки клиентов актуальны.
- [ ] Убедиться: удалённый заказ снова доступен в «Izvēlēties pasūtījumus» для других рейсов (статус confirmed, `trip_id` = null).

## 5. Документы

### 5.1 CMR по грузу

- [ ] На странице рейса в блоке груза указать номер CMR, нажать «Ģenerēt» CMR.
- [ ] Проверить: PDF создаётся и сохраняется; ссылка на скачивание/открытие работает; в CMR отображаются рейс, шаги, отправитель/получатель, позиции груза, вес/объём.
- [ ] Груз без позиций (items) — генерация не должна выполняться (ошибка валидации).

### 5.2 Invoice (rēķins) по грузу

- [ ] Указать номер счёта, нажать генерацию инвойса.
- [ ] Проверить: PDF счёта создаётся; сумма, НДС, реквизиты экспедитора и данные груза отображаются корректно.
- [ ] При наличии Dikstāve (задержка): дни и сумма отображаются в инвойсе; снятие галочки/удаление простоя обновляет инвойс (инвалидация PDF).

### 5.3 Transport Order (pasūtījums) по грузу

- [ ] Сгенерировать документ «Pasūtījums» по грузу — PDF создаётся и открывается/скачивается.

### 5.4 Документы по шагам и по рейсу

- [ ] Загрузить документ по шагу (тип, комментарий, файл JPG/изображение) — документ сохраняется, отображается в блоке шага.
- [ ] Удалить документ по шагу — запись и файл удаляются.
- [ ] Загрузить общий документ по рейсу — отображается в соответствующем блоке.

## 6. Таблицы и списки

### 6.1 Список рейсов

- [ ] Открыть `/trips` — таблица рейсов загружается; колонки (номер, даты, экспедитор, водитель, статус и т.д.) соответствуют данным.
- [ ] Фильтры по дате, статусу (если есть) — результат фильтрации корректен.
- [ ] Пагинация — переключение страниц работает, количество записей ожидаемое.

### 6.2 Список заказов

- [ ] Открыть `/orders` — отображаются все заказы; колонки (номер, дата, экспедитор, заказчик, статус, рейс) корректны.
- [ ] Заказ со статусом «Pārvērts reisā» — отображается ссылка на рейс; редактирование заказа недоступно.

### 6.3 Просмотр рейса — блоки данных

- [ ] Блок «Reisa maršruts»: все шаги в правильном порядке; при нескольких грузах привязка грузов к шагам видна (если реализовано в UI).
- [ ] Блок «Kravas»: все грузы с позициями, ценами, НДС; контейнер/пломба, TIR — если заполнены.
- [ ] Блок «Pasūtījumi reisā»: все привязанные заказы; при нескольких грузах в заказе — аккордеон или список по каждому грузу (отправитель/погрузка | получатель/разгрузка).
- [ ] Блок «Izvēlēties pasūtījumus»: селект с многострочными опциями (номер заказа, заказчик, по каждому грузу строка отправитель/дата — получатель/дата); выбор нескольких заказов и кнопка «Pievienot pasūtījumus reisam» работают.
- [ ] Расходы по рейсу, события одометра — отображение и добавление (если тестируем полный сценарий).

## 7. Автоматизированные тесты (существующие и рекомендуемые)

### Уже есть

- `CreateTripTest`: создание рейса со шагами и грузами (фабрики).
- `CreateTripComponentTest`: Livewire CreateTrip — пересчёт НДС; валидация порядка шагов (разгрузка до погрузки).
- `CreateTripFailsTest`: нельзя создать рейс, если разгрузка раньше погрузки (POST валидация).
- `TripFullScenarioTest`: сценарий с несколькими шагами и грузами, привязка по ролям loading/unloading.
- Unit: `TripModelTest`, `TripCargoTest`, `TripStepTest`.

### Рекомендуется добавить

1. **CreateTripFromOrderTest** (Feature): создание рейса с предзаполнением из заказа — создание TransportOrder + OrderStep + OrderCargo (или сидер/фабрика), вызов CreateTrip с `from_order_ids`, сохранение — проверка, что TripStep и TripCargo созданы и заказ в статусе converted с `trip_id`.
2. **ViewTripAddRemoveOrderTest** (Feature): рейс с одним привязанным заказом; вызов `addOrdersToTrip` с ещё одним заказом — проверка появления шагов и грузов; вызов `removeOrderFromTrip` — проверка удаления шагов/грузов и сброса заказа в confirmed без trip_id.
3. **TripDocumentsTest** (Feature): рейс с грузом и позициями; генерация CMR (мок Storage/PDF) — проверка, что файл создаётся и запись обновляется; при отсутствии позиций — ошибка валидации.
4. **EditTripTest** (Feature): создание рейса, редактирование шагов и грузов через EditTrip, сохранение — проверка изменений в БД и отсутствия потери связей.

- [ ] При отсутствии колонки `transport_order_id` страница просмотра рейса не падает (проверка Schema::hasColumn в коде).

## 10. Запуск автотестов

В корне проекта (Fleet или Cargo Trans):

```bash
php artisan test
php artisan test tests/Feature/TripAndOrdersScenarioTest.php
php artisan test tests/Feature/CreateTripTest.php tests/Feature/CreateTripComponentTest.php tests/Feature/TripFullScenarioTest.php
```

Сценарии из раздела 7 (CreateTripFromOrderTest, ViewTripAddRemoveOrderTest и т.д.) частично покрыты в `tests/Feature/TripAndOrdersScenarioTest.php`: просмотр рейса, страница создания, добавление заказов к рейсу, пустой выбор, удаление заказа из рейса.
